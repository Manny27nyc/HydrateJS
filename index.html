HydrateJS by nanodeath

Make javascript object serialization a snap! Have you found JSON.stringify and JSON.parse insufficient? HydrateJS may be what you're looking for. Note: designed mainly for browsers, but may have a place with server-side code as well. Untested though.

Native JSON serialization (via JSON.stringify) is convenient, but has a couple critical pitfalls. For one, if your Javascript object is actually an instance of something other than a hash ({}), this information is lost when serializing -- so a method that may have existed pre-serialization on an object won't exist post-deserialization. Another problem is circular references -- they simply don't work. Lastly, object references -- i.e. two elements in an array that point to the exact same object -- aren't deserialized as one might expect. HydrateJS aims to fix all that. Other things native JSON handling couldn't hope to provide that I hope to fill: things like migrations. After you deserialize something that has been stored away in a database for a month, the class on which it was based may have changed -- json migrations help you deal with that.

Dependencies

Only json2.js, but this is bundled with HydrateJS and automatically required into the page as needed.

Install

HydrateJS is written in CoffeeScript, a new spiffy language that compiles down to javaScript. You can get started here: http://www.coffeescript.com/ (you'll need to install Node and npm if you haven't already, but that should take no more than 5 minutes). Then run ./compile.sh. If I can figure out a way to automatically generate the .js so you don't have to, I will.
